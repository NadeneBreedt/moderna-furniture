Priority Fixes From The Live Site Test

These are the first things I would clean up before a visual redesign, because they directly affect enquiries and trust.

High

Category pages break

Direct category URLs can show the heading but no products because `loadProducts` hits a missing element listener.

High

JS errors on every route

The homepage script calls `addEventListener` on an element that is not present, which can stop later behavior.

Medium

WhatsApp placeholder

Product detail pages link to `wa.me/your_number`, which should be replaced or removed.

Medium

Hidden search label

The header search link has no accessible label, so it appears as an unnamed link in testing.

What I’d Pick First

My recommendation is a practical hybrid: fix the technical issues, then move toward the Catalogue-Led direction with stronger project proof.

Phase 1: make it work

  • Fix category product loading.
  • Remove duplicate Supabase imports/declarations.
  • Give search and quote controls proper labels.
  • Replace the WhatsApp placeholder.

Phase 2: make it sell

  • Homepage search bar and quote CTA above the fold.
  • Category cards for the most profitable B2B ranges.
  • Product spec layout with quote basket always visible.
  • Customer proof and installation/warranty blocks near CTAs.
